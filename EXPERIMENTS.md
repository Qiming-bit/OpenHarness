# OpenHarness 魔改实验任务

> 每个阶段对应 LEARNING_PLAN.md 的学习内容，完成实验即可算作掌握。
> 难度：★ 简单 / ★★ 中等 / ★★★ 进阶

---

## 实验 1：环境搭建验证（对应阶段一）★

**目标**：确认环境就绪，理解 CLI 的完整启动流程。

**任务**：
1. 安装依赖并成功启动 `oh` 交互模式
2. 使用 `--dry-run` 模式预览配置
3. 尝试不同 Provider（Anthropic / OpenAI / Ollama），记录配置差异
4. 画出 CLI 启动到 Agent Loop 开始的调用链

**验证标准**：能说出 `oh` 命令背后经历了哪些模块的初始化。

---

## 实验 2：自定义 Tool 开发（对应阶段三）★★

**目标**：理解 ToolRegistry 的注册、执行、权限校验流程。

**任务**：
1. 在 `src/openharness/tools/` 下新建一个自定义 Tool（例如：`jira_tool` 查询 Jira 工单）
2. 实现 Tool 的输入参数校验、API 调用、结果格式化
3. 在 ToolRegistry 中注册
4. 配置该 Tool 的权限（需要用户审批才能执行）
5. 通过 `oh` 对话调用该 Tool，验证端到端流程

**进阶**：实现一个 MCP Tool 集成（对接外部 MCP Server）

**验证标准**：对话中 Agent 能发现并调用你的自定义 Tool，权限审批流程正常。

---

## 实验 3：自定义 Skill 开发（对应阶段五）★

**目标**：理解 Markdown Skill 格式和按需加载机制。

**任务**：
1. 在 `.claude/skills/` 或 `src/openharness/skills/` 下创建一个新 Skill
2. Skill 内容自选一个实用场景（如：代码审查检查清单、Git 提交规范）
3. 验证 Skill 在对应场景下被自动激活
4. 对比 anthropics/skills 格式和 Claude-style 格式的差异

**验证标准**：在相关对话场景中，Skill 内容被正确注入到上下文中。

---

## 实验 4：Memory 系统定制（对应阶段四）★★

**目标**：理解 MEMORY.md / CLAUDE.md 的自动发现和注入机制。

**任务**：
1. 在子目录中创建 CLAUDE.md，验证自动发现是否生效
2. 修改 MEMORY.md，添加一条 user 类型记忆，下次对话验证是否被读取
3. 实验：在不同层级的目录中创建 CLAUDE.md，观察优先级
4. 分析 auto-compact 发生时，哪些记忆被保留、哪些被丢弃

**验证标准**：能解释记忆从写入到读取的完整链路。

---

## 实验 5：Subagent 实战（对应阶段六）★★★

**目标**：理解多 Agent 协同的生成、执行、结果回传。

**任务**：
1. 构造一个需要并行处理的任务（例如：同时搜索 3 个目录的文件并汇总）
2. 通过 Agent 工具生成 subagent，观察任务拆分策略
3. 记录 subagent 的执行日志和结果回传过程
4. 实验：构造 subagent 冲突场景（两个 subagent 同时修改同一文件），观察行为
5. 尝试修改 subagent 的上下文隔离策略

**验证标准**：能画出 subagent 的生命周期图，解释上下文隔离的实现方式。

---

## 实验 6：渠道适配器分析（对应阶段七）★★

**目标**：理解多渠道适配器的设计模式。

**任务**：
1. 选择 3 个渠道实现（如 Telegram / 飞书 / Discord），对比代码结构
2. 提取各渠道的公共抽象，画出适配器模式的类图
3. 实验：实现一个极简的渠道适配器（如：stdout 输出）
4. 分析 Bridge 如何管理多渠道的会话状态映射

**验证标准**：能解释为什么用适配器模式，以及新增一个渠道需要做哪些工作。

---

## 实验 7：Hook 自定义（对应阶段八）★★

**目标**：理解事件驱动钩子系统的扩展方式。

**任务**：
1. 分析现有 Hook 的实现（pre-tool / post-tool / on-error）
2. 实现一个自定义 Hook：每次工具调用后记录耗时日志
3. 实现一个自定义 Hook：禁止执行包含 `DROP` 的 SQL 命令
4. 实验：Hook 的执行顺序如何控制？能否让某个 Hook 优先执行？

**验证标准**：自定义 Hook 能正确注册、触发、执行。

---

## 实验 8：ohmo Personal Agent 定制（对应阶段九）★

**目标**：理解个人助理应用的上层定制。

**任务**：
1. 修改 soul.md，定义 ohmo 的人格
2. 修改 user.md，添加你的用户画像
3. 体验 Gateway 多渠道接入（如接入 Telegram）
4. 对比 ohmo 和 OpenHarness 核心能力的边界

**验证标准**：能解释"框架 vs 应用"的边界划分。

---

## 实验 9：Autopilot 定时任务（对应阶段六）★★

**目标**：理解自动化调度的 Cron 机制。

**任务**：
1. 创建一个定时任务（如：每天凌晨检查 git 状态）
2. 观察 Autopilot 的调度日志
3. 分析 Cron 表达式解析和任务触发的实现
4. 实验：创建一个事件触发的自动化任务

**验证标准**：定时任务能按预期触发并执行。

---

## 实验 10：综合项目（对应阶段十）★★★

**目标**：综合运用所学知识，完成一个完整的功能扩展。

**任务**（任选其一）：
1. **实现一个完整的 Plugin**：包含自定义 Tool + Skill + Hook，形成一个小功能集
2. **渠道 + Subagent 组合**：通过 Telegram 接收任务，拆分为 subagent 执行，结果回传到 Telegram
3. **自定义 Provider**：对接一个 OpenAI 兼容的本地模型（如 Ollama），验证 Agent Loop 适配

**验证标准**：端到端功能可用，代码结构符合项目规范。
